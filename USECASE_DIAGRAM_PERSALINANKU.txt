RELASI ANTAR USE CASE:
==========================================

INCLUDE RELATIONSHIP:
- Login → Validasi kredensial
- Tambah data → Validasi input
- Upload file → Kompresi gambar
- Generate laporan → Export data
- Chat → Enkripsi pesan

EXTEND RELATIONSHIP:
- Login → Dashboard (jika berhasil)
- Tambah data → Notifikasi (jika berhasil)
- Upload file → Backup (jika berhasil)
- Chat → Log aktivitas
- Darurat → Notifikasi bidan

GENERALIZATION:
- Admin → Bidan (spesialisasi)
- Pasien → User (generalization)
- Notifikasi → Push, Email, SMS
- Data → Pasien, Persalinan, Pemeriksaan

==========================================
KESIMPULAN:
==========================================

Use Case Diagram ini menunjukkan bahwa aplikasi PersalinanKu memiliki dua aktor utama dengan kebutuhan yang berbeda. Bidan (Admin) memiliki akses ke fitur administratif yang komprehensif termasuk manajemen sistem, sementara Pasien memiliki akses ke fitur layanan yang user-friendly. Semua fitur sistem diintegrasikan ke dalam aktor Bidan untuk memudahkan pengelolaan dan maintenance aplikasi.
